            <div class="docstring"><p>The <code>wx.App</code> class represents the application and is used to:</p>

<ul>
<li>bootstrap the wxPython system and initialize the underlying
gui toolkit</li>
<li>set and get application-wide properties</li>
<li>implement the native windowing system main message or event loop,
and to dispatch events to window instances</li>
<li>etc.</li>
</ul>

<p>Every wx application must have a single <code>wx.App</code> instance, and all
creation of UI objects should be delayed until after the <code>wx.App</code> object
has been created in order to ensure that the gui platform and wxWidgets
have been fully initialized.</p>

<p>Normally you would derive from this class and implement an <code><a href="#CoroApp.OnInit">OnInit</a></code>
method that creates a frame and then calls <code>self.SetTopWindow(frame)</code>,
however <code>wx.App</code> is also usable on its own without derivation.</p>

<p>:note: In Python the wrapper for the C++ class <code>wxApp</code> has been renamed tp
    :class:<code>wx.PyApp</code>. This <code>wx.App</code> class derives from <code>wx.PyApp</code>, and is
    responsible for handling the Python-specific needs for bootstrapping the
    wxWidgets library and other Python integration related requirements.</p>
